# Универсальный промпт аудита — template-starter

Скопируй блок ниже в Composer или запусти команду **Run audit** из `.cursor/commands/`.

---

## Контекст

Ты проводишь **полный аудит репозитория template-starter** по правилам из `audit/rules.json` и согласованно с `CLAUDE.md`.

## Обязательные шаги

1. Прочитай `audit/rules.json` (все записи в `rules`).
2. Пройди по кодовой базе (`src/app/`, `src/components/`, `src/lib/`, `src/hooks/`, `src/types/`, корневые конфиги, `.github/workflows/`) и для **каждого** правила зафиксируй статус:
   - `pass` — соблюдено
   - `fail` — нарушение (укажи файлы и строки/фрагменты)
   - `na` — не применимо к этому проекту (кратко почему)
3. Если два правила конфликтуют между собой или с `CLAUDE.md`, **не молчи**: предложи разрешение и добавь запись в `audit/conflicts.log` (формат см. в файле).
4. Сформируй отчёт в Markdown:
   - Резюме (2–4 предложения)
   - Таблица: Rule ID | Severity | Status | Комментарий
   - Список **CRITICAL** и **HIGH** в порядке приоритета исправления
5. **Обновление базы правил:** если в ходе аудита выявлены новые повторяющиеся проблемы, предложи **до 5** новых правил в том же JSON-формате, что существующие элементы `rules` (id, title, severity, category, description, verify, autoFixable). Не дублируй уже существующие id.
6. **История:** добавь одну запись в `audit/audit-history.json` в массив `entries`:

```json
{
  "id": "<uuid или короткий уникальный id>",
  "startedAt": "<ISO8601>",
  "finishedAt": "<ISO8601>",
  "summary": "<одно предложение>",
  "findings": { "critical": 0, "high": 0, "medium": 0, "low": 0, "na": 0 },
  "rulesUpdated": false,
  "notes": "<опционально: что меняли в rules.json>"
}
```

7. Если пользователь явно просит обновить `audit/rules.json` новыми правилами — внеси изменения аккуратно, увеличь `meta.ruleCount`, выстави `meta.lastReviewed` в сегодняшнюю дату (ISO), сохрани валидный JSON.

## Ограничения

- Не выдумывай содержимое секретов; не коммить реальные ключи.
- После правок кода запусти `pnpm lint` и `pnpm tsc --noEmit`, если доступно.

---

## Короткая версия (одной строкой для вставки)

Проведи полный аудит по `audit/rules.json`, выдай отчёт с таблицей статусов, обработай конфликты правил в `audit/conflicts.log`, допиши запись в `audit/audit-history.json`, при согласии пользователя обнови `audit/rules.json` новыми устойчивыми правилами.
