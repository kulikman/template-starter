## История

### 2026-04-26 — Пустой push при git rm -rf + tar xzf
**Что сломалось:** После `git rm -rf .` и `tar xzf archive -C .` репо оказалось пустым — коммит прошёл с 0 файлов.
**Причина:** `tar xzf` без `--strip-components` создал вложенную папку внутри репо вместо распаковки в корень.
**Как починили:** Клонировать репо, скопировать файлы напрямую через `cp -r`, затем `git add -A`.
**Как не допустить:** При распаковке архива в существующую директорию всегда проверять `ls` после `tar xzf`. Использовать `cp -r` вместо `tar` когда файлы уже доступны локально.
